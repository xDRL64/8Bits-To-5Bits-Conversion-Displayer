Change 5 bits color channel values : Update 5 bits color preview Update 8 bits color preview (by using "lavanda" 5bits to 8bits table) Using : This tool is mainly made to observe behavior of convertion from 8bits to 5bits. Every conversion is done with "pre-made" tables. 8bits to 5bits tables contain 256 cells and each cell can hold a value between 0 and 31. It can use arbitrary logic to hard code a convertion table or it can use a "for loop" from 0 to 255 to make tables in which each cell is result of a calc process at each loop step. Tables are displayed in colored columns. In this prototype version the 2 red columns and the 2 orange columns are got by calcs with simple formulas and the 2 blue columns are got by more complex logical process. Each convertion table has a green statistic column at its right side. Statistic colums have 32 cells and each cell has an index between 0 and 31 and so each cell displays the number of times the value that corresponds to its cell index is provided by its corresponding conversion table. Thus a statistic column shows the result patern of its convertion table. The white column is an index as visual cue for every other columns. The violet (lavanda) column is the table for conversion from 5bits to 8bits. It is hard coded, but based on SNES colors displayed on emulator. (ripped from SGM2 palette editor) And it is used in the building of the 2 blue table columns. Observe Reversibility Tables : Notice the reversibility or the non-reversibility of some conversion tables. What I name "Reversibility" of tables is "constant synchronized loop" in convert tables "linking relation". On this next image, see the multiple and successive non-reversibilities. From the index : 17, table cell holds value : 2 (5bits) By tracing relation from [this 8bits to 5bits table] to [the 5bits to 8bits table] the non-reversibilities is observable by seeing this "desynchronization" in the revert table relation. (Arrow colors do not matter, it is just for lisibility) (Keep focus on the orange and the violet column to figure out) While column selection by click on it, it will display its reversibility state at top left of the window : "Sync" on green or "Not Sync" in red It does not precise the level of non-reversibility. If it finds only one level of non-reversibility this considers it as non-reversible. Reversibility is also checked and displayed while holding SHIFT + CTRL + Mouse Wheel movement on some special columns. (These special columns allow to rebuild them once that their variable parameter has changed.) Special table columns with updatable parameter : In this prototype version the second red column, the second orange colum, and the second blue column are special columns. They allow to rebuild them with a variable parametre that change while SHIFT + CTRL + Mouse Wheel movement. Special columns are mentionned on CTRL + Mouse Enter over a column. On every update it will display, the new value for its parameter and its reversibility state, at top left of the window.
